# 31st File ERP 

A full-stack financial and compliance management application featuring automated journal entries, live reporting (Trial Balance, P&L, Balance Sheet), and UPI capture capabilities. 

## 📦 Project Structure
- **Frontend:** 11 pixel-matched screens (React/Next.js)
- **Backend:** 5 REST API routes 
- **Database:** PostgreSQL (hosted via Neon.tech)
- **Auth:** NextAuth

## 🚀 Deployment & Setup Guide (Replit)

Follow these steps to deploy or migrate the application to a new Replit workspace.

### Step 1: Upload Files
1. Create a new Repl (choose Node.js or Next.js template).
2. Drag and drop `31st-file-erp.zip` into the file panel.
3. Open the Shell and unzip the project:
   ```bash
   unzip 31st-file-erp.zip

```
### Step 2: Configure Environment Variables
Navigate to the **Secrets (🔒)** tab in Replit and add the following keys:
| Secret Name | Value | Description |
|---|---|---|
| DATABASE_URL | postgresql://... | Connection string from your free Neon.tech account. |
| NEXTAUTH_SECRET | *(Generated string)* | Run openssl rand -base64 32 in the Shell and paste the output here. |
| NEXTAUTH_URL | https://your-repl-name.replit.app | The public web address of your current Repl. |
### Step 3: Initialize Database & Run
Open the Shell tool and execute the following commands in order:
```bash
# 1. Install all dependencies
npm install

# 2. Push the database schema to Neon (creates the 5 core tables)
npm run db:push

# 3. Seed the database with the 30 default Chart of Accounts
npm run db:seed

# 4. Start the development server
npm run dev

```
### Step 4: Verify Live Status
Once the server starts on port 3000, navigate to the webview.
 1. Create a test UPI entry.
 2. Post the entry.
 3. Verify that the Trial Balance, P&L, and Balance Sheet update dynamically.
